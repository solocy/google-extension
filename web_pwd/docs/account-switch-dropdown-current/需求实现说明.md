# WebPwd 切换账号下拉与当前账号标识需求实现说明

## 需求目标
- 不使用弹窗，切换账号仍采用右上角下拉方式。
- 下拉中展示当前登录页面的全部账号。
- 为当前已自动补全的账号增加明确标识。
- 登录动作发生后，切换账号下拉入口自动消失。

## 实现方案
- 在 `content_script.js` 中保留右上角下拉入口，不再使用弹窗结构。
- 新增当前补全账号标识状态 `currentAutoFilledCredentialKey`，用于记录当前页面默认或手动切换后的账号。
- 下拉菜单渲染全部账号列表，不再只显示非第一个账号。
- 当前已补全账号在下拉项中显示 `当前 / Current` 标记。
- 默认自动填充统一通过 `applyDefaultCredential` 执行，确保当前账号标识与页面实际填充状态一致。
- 登录提交、登录按钮点击、页面离开时仍会清理下拉入口。
- 对插件自身注入的切换账号按钮和保存卡片按钮增加点击隔离，避免被页面登录按钮监听误判。

## 影响文件
- `web_pwd/content_script.js`

## 验收要点
1. 同一登录页保存多个账号时，页面右上角显示“切换账号”按钮。
2. 点击按钮后以下拉形式展示当前登录页全部账号。
3. 默认自动填充的账号在下拉中带有“当前”标识。
4. 点击其他账号后页面切换填充，并更新“当前”标识。
5. 点击登录后，下拉入口消失。
6. 点击“切换账号”按钮时，下拉不会立即自行关闭。

## 回滚说明
- 如需回滚，可将 `showAccountSwitcher` 恢复为仅展示部分账号的旧逻辑，并移除 `currentAutoFilledCredentialKey` 标识状态。

# WebPwd 登录页精确匹配、多语言与多账号切换需求实现说明

## 需求目标
- 登录凭证按完整登录页面 URL 匹配，而不是仅按域名匹配。
- 同一登录页面下允许保存多个账号。
- 同一登录页面下，同一账号仅保留一个密码；当密码变化时，提示用户是否更新密码。
- 插件管理页在鼠标悬浮时可展示完整登录网址。
- 增加语言切换，支持中文和英文，默认中文。
- 同一登录页面有多个账号时，自动填充默认使用第一个账号，并在页面右上角提供其他账号的精简切换入口。

## 实现方案
### 1. 后台匹配与存储规则
- 在 `background.js` 中新增登录页 URL 规范化逻辑，保存和匹配时统一按完整 URL（去掉 hash）处理。
- 凭证冲突判定从“同域名同账号”调整为“同登录页 URL 同账号”。
- 保存逻辑调整为 upsert：若同登录页同账号已存在，则直接更新这条记录，避免同账号产生多条密码记录。
- 新增 `findCredentialsForPage` 消息接口，返回当前登录页的全部账号列表及设置项。

### 2. 页面内提示卡片
- `content_script.js` 中的保存提示卡片根据后台返回的 `hasConflict` 动态展示“保存登录信息”或“更新密码”。
- 卡片文案接入中英文词典，语言由扩展设置驱动。
- 相同账号同密码时不弹窗；相同账号不同密码时提示更新；不同账号时提示保存新账号。

### 3. 多账号自动填充与切换
- 页面自动填充时，从当前登录页匹配到的账号列表中默认取第一个账号进行填充。
- 如果同一登录页存在多个账号，则在页面右上角注入一个精简切换入口，仅展示账号名供用户选择。

### 4. 插件管理页增强
- 在 `popup.html` / `popup.js` 中增加语言切换下拉框。
- 管理页的按钮、提示语、输入占位符接入中英文文案。
- 登录记录中增加登录页 URL 预览行，悬浮时使用浮层展示完整 URL。
- 默认文件夹在英文界面下显示为 `Default`，不修改底层存储值。

## 影响文件
- `web_pwd/background.js`
- `web_pwd/content_script.js`
- `web_pwd/popup.html`
- `web_pwd/popup.js`
- `web_pwd/styles.css`

## 配置项说明
- `autoFillEnabled`: 是否启用自动填充，布尔值。
- `language`: 界面语言，`zh` 或 `en`，默认值为 `zh`。

## 验收要点
1. `https://local.console.gov-bid.com/bbiao-admin/login` 与 `https://local.console.gov-bid.com/bbiao-user/login` 分别保存账号后，互不串用。
2. 同一登录页面下相同账号相同密码，不再重复弹窗。
3. 同一登录页面下相同账号不同密码，提示更新密码。
4. 同一登录页面下不同账号，允许继续保存。
5. 自动填充时默认填充该页面保存的第一个账号。
6. 当同一登录页面有多个账号时，页面右上角出现其他账号切换入口。
7. 插件管理页切换语言后，主要文案可切换为英文。
8. 插件管理页悬浮登录记录标题或登录页 URL 时，可看到完整登录地址。

## 回滚说明
- 如需回滚，可恢复 `background.js` 中按域名匹配的逻辑、移除 `findCredentialsForPage` 接口，并回退 `content_script.js` 和 `popup.js` 的多语言与多账号切换逻辑。

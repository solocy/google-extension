# WebPwd 登录页精确匹配、多语言与多账号切换上线操作说明

## 配置项说明
### 新增/调整配置
- `language`
  - 说明：扩展界面语言。
  - 可选值：`zh`、`en`
  - 默认值：`zh`
- `autoFillEnabled`
  - 说明：是否启用自动填充。
  - 可选值：`true`、`false`
  - 默认值：`true`

## 本地检查命令
```powershell
Set-Location d:\project\google-extension
Get-Item .\web_pwd\background.js
Get-Item .\web_pwd\content_script.js
Get-Item .\web_pwd\popup.js
```

## Chrome 扩展刷新步骤
1. 打开 `chrome://extensions/`
2. 开启“开发者模式”
3. 在 `web_pwd` 扩展上点击“重新加载”
4. 刷新目标登录页后重新验证

## 建议验证脚本
```powershell
Set-Location d:\project\google-extension
Write-Output '1. 重新加载 web_pwd 扩展'
Write-Output '2. 分别打开 admin/login 与 user/login 页面保存不同账号'
Write-Output '3. 确认两个登录页互不串用账号'
Write-Output '4. 同页保存同账号新密码时，确认弹出更新密码提示'
Write-Output '5. 同页存在多个账号时，确认默认填充第一个，并在右上角展示其他账号入口'
Write-Output '6. 打开插件管理页，切换 Language 为 English，检查主要文案是否切换'
```

## 打包命令
```powershell
Set-Location d:\project\google-extension
Compress-Archive -Path .\web_pwd\* -DestinationPath .\web_pwd-login-page-matching-i18n.zip -Force
```

## 上线风险提示
- 若已有非常老的历史数据仅保存了域名而未保存完整 URL，可能仍会按域名作为兼容兜底匹配。
- 多账号切换入口为页面注入 UI，建议验证典型登录页、Ajax 登录页以及响应式页面布局场景。
- 语言切换当前覆盖扩展管理页和页面注入卡片，建议分别做中英文回归验证。

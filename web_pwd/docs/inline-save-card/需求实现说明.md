# WebPwd 页面内滑出保存卡片需求实现说明

## 需求目标
- 将原有独立弹窗式保存提示改为页面内固定卡片。
- 卡片从页面右下角沿右侧滑入。
- 卡片展示 10 秒后自动消失。
- 保留手动关闭、保存到文件夹、编辑标题、冲突提示等能力。

## 实现方案
- 改造内容脚本 `content_script.js` 中的登录检测回调，不再调用后台创建独立 popup window。
- 在当前页面内动态注入样式和 DOM 结构，渲染右下角固定卡片。
- 使用 CSS 动画实现右侧滑入和滑出效果。
- 使用 10000ms 自动关闭定时器，卡片无操作时自动消失。
- 保存动作改为直接调用 `storeCredential` 消息，等待后台返回结果后给出 toast 提示。

## 影响范围
- 影响文件：`web_pwd/content_script.js`
- 未修改存储结构。
- 未修改凭证保存、冲突判断、自动填充的数据协议。
- 后台中原有 popup prompt 代码暂未删除，不影响当前新流程。

## 验收要点
1. 在任意登录页提交账号密码后，页面右下角出现卡片。
2. 卡片以右侧滑入方式出现，不再打开独立浏览器小窗口。
3. 无操作时 10 秒后自动消失。
4. 点击“保存”后记录成功，并提示“登录信息已保存”。
5. 点击“取消”或关闭按钮后卡片消失。
6. 检测到同账号不同密码时显示覆盖提醒。

## 回滚说明
- 如需回滚，可将 `content_script.js` 中登录检测后的页面内卡片逻辑恢复为调用 `showSaveNotification` 后台消息。

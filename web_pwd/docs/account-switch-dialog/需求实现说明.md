# WebPwd 切换账号弹窗需求实现说明

## 需求目标
- 将页面右上角原有的“其他账号”下拉菜单改为“切换账号”弹窗。
- 弹窗展示当前登录页面下保存的全部账号。
- 点击账号后立即填充该账号。
- 登录动作发生后，切换账号入口和弹窗自动消失。

## 实现方案
- 在 `content_script.js` 中新增 `ACCOUNT_SWITCHER_DIALOG_ID`，把原有下拉菜单重构为蒙层弹窗。
- 切换账号入口保留在页面右上角，按钮文案改为“切换账号”。
- 弹窗中展示当前登录页的全部账号名，包括默认自动填充的第一个账号。
- 点击某个账号后，调用现有填充逻辑 `fillCredentialDirectly` 完成切换。
- 在表单提交、登录按钮点击、页面离开时统一调用 `removeAccountSwitcher()`，确保登录完成后入口和弹窗消失。

## 影响文件
- `web_pwd/content_script.js`

## 验收要点
1. 同一登录页保存多个账号时，页面右上角显示“切换账号”按钮。
2. 点击“切换账号”后，弹出账号选择弹窗。
3. 弹窗中可看到当前登录页的全部账号。
4. 点击任一账号后，对应账号和密码被填充到页面。
5. 点击登录提交后，切换账号按钮和弹窗消失。
6. 页面跳转或刷新后，切换账号弹窗不会残留。

## 回滚说明
- 如需回滚，可将 `content_script.js` 中 `showAccountSwitcher` 恢复为下拉菜单实现，并去掉登录提交时的统一清理逻辑。
